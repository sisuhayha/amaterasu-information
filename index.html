About the Project

What is Amaterasu?

An autonomous AI agent that independently plans, executes and continuously improves its approach to any goal.

Amaterasu is a complete Python framework for autonomous AI agents. The agent receives a goal, decomposes it via HTN Planner into subtasks, executes them with registered tools and evaluates every step in a ReAct loop.

The system supports 5 LLM backends (Ollama, Claude, Groq, Gemini, Free) with intelligent semantic routing — each task is automatically forwarded to the best model for the job.

Self-Improvement allows the agent to analyze its own code and generate safe patches — backed by 12 security guarantees and an explicit opt-in requirement.

In idle mode the Dream Mode activates: the agent analyzes past episodes, recognizes patterns, generates new task ideas and optimizes strategies autonomously.

System Flow

  • 01
    Goal Input & Clarification GoalClarifier, ComplexityEstimator, ReasoningBudget
  • 02
    Adaptive Planning HTN Planner → CheckpointPlanner → DAG with subtasks
  • 03
    ReAct Loop Think → SafetyGate → Execute → Reflect
  • 04
    Learning & Persistence SkillEvolution, StrategyLibrary, KnowledgeGraph
  • 05
    Dream Mode Pattern recognition, strategy optimization, idea generation
  • 06
    Shutdown & Handover Session document, memory consolidation, WorldState save

What Amaterasu Can Do

Core Capabilities

Amaterasu acts as a fully autonomous digital employee — it can research, code, automate, analyze and continuously learn without human intervention.

🔍

Deep Research & Analysis

Amaterasu crawls the web, ingests PDFs and documents, builds a knowledge graph and retrieves relevant information through RAG pipelines. It synthesizes findings into structured reports autonomously.

Code Generation & Execution

From writing clean Python to running tests and debugging, Amaterasu handles the full development loop. It uses Git tools (12 included), executes code in sandboxed environments and commits changes automatically.

Python Executor · Git · 14+ Tools
🤖

Multi-Agent Orchestration

Deploy specialized sub-agents in parallel: PlannerAgent, ResearcherAgent, CoderAgent, CriticAgent, ExecutorAgent. The SwarmOrchestrator coordinates them and a Debate Mode resolves complex decisions through agent consensus.

Swarm · Debate · Coordinator

Persistent Memory & Learning

Working memory (RAM), episodic memory (SQLite + FTS5), semantic memory and FAISS vector store work together via cascading retrieval. Amaterasu remembers every past episode and applies learned strategies to new tasks.

FAISS · SQLite · Embeddings

Desktop & Browser Automation

Control any GUI application via pyautogui, capture screenshots, perform OCR with Tesseract and automate browser tasks. Amaterasu can interact with graphical interfaces that have no API.

pyautogui · OCR · Browser

8 independent security layers: capability-based access control, anti-tampering (SHA256), global kill-switch, per-subsystem circuit breakers, outbound whitelist, eBPF anomaly monitor and human-in-the-loop approval for high-risk actions.

System Flow — ReAct Loop in Detail

Input goal <text> → GoalClarifier resolves ambiguities → ComplexityEstimator determines tier: simple / medium / complex / extreme
Planning AdaptivePlanner (instant / quick / standard / deep) → CheckpointPlanner decomposes into checkpointed steps with DAG dependencies
Think LLM decides next action using StrategyLibrary, BeliefStore and ReasoningBudget
Safety Gate Score 1–4: auto-approved · Score 5–6: simulated · Score 7: Simulator shows consequences · Score ≥ 8: manual approval required · Score 10: blocked
Execute ToolExecutor with adaptive timeout, retry logic, EpisodicMemory persistence and WorldState update
Reflect ReflectionEngine evaluates result → decision: continue / retry / replan / finish / escalate

Standout Features

What makes Amaterasu unique

01

Semantic Model Routing

The router uses sentence-transformers to semantically analyze each task and route it to the optimal backend. Claude for reasoning, Groq for speed, Gemini for research. Human-readable routing explanation via route_with_reason().

02

Self-Improvement with 12 Guarantees

The agent can analyze its own code and generate patches. Every patch goes through: Backup, Syntax Check, Import Test, Rollback mechanism, Audit Log, Rate Limit (5 patches/hr), Kill-Switch, Circuit Breaker and Capability Check.

03

Dream Mode (Idle Reflection)

After 15 seconds of idle, Amaterasu automatically analyzes past episodes, recognizes behavioral patterns, generates new task ideas and optimizes strategies. The result is persisted as a structured Dream Report.

04

Multi-Layer Security Architecture

8 independent security layers: Capability-based access control, Anti-Tampering with SHA256 baseline, global Kill-Switch, Circuit Breaker per subsystem, Outbound Whitelist and eBPF Anomaly Monitor at Python level.

05

MCP Server Integration

Amaterasu can be embedded as an MCP tool in Claude Desktop and Cursor — via SSE or stdio. P2P networking of multiple Amaterasu instances via PeerRegistry and MCPPeerClient.

06

Checkpoint Planner & Replay

On crash, execution resumes from the last checkpoint — not from the start. Replay Mode shows each step as a timeline with reasoning, tool call and result — ideal for debugging and audit.

07

Skill Evolution Engine

Skills with under 60% success rate are automatically mutated and improved. The Skill Composer recognizes frequent skill pairs from episode history and combines them into pipeline skills. Marketplace with rating system.

08

Human-in-the-Loop

Critical actions (Safety Score ≥ 8) require manual confirmation with diff view. Mission Templates for common workflows: code_review, bug_hunt, repo_refactor, research, security_audit — ready to deploy.

Get Started

Ready in minutes

Python 3.11+ and Ollama for local LLM are recommended. All LLM backends are optional.

Installation
# Clone repository git clone https://github.com/yourname/amaterasu cd amaterasu # Minimal install pip install -e . # Full install (recommended) pip install -e ".[all]" # Pull Ollama model (local) ollama pull llama3
API Keys (.env)
# Never in YAML — only in .env ANTHROPIC_API_KEY=sk-ant-... GROQ_API_KEY=gsk_... GOOGLE_API_KEY=AIza... AMATERASU_API_TOKEN=my-web-token
Run
# Interactive CLI mode python amaterasu.py --interactive # Run a single goal python amaterasu.py --goal "Research GPU prices 2025" # Autonomous mode python amaterasu.py --autonomous \ --goal "Analyze my Python project" # Web UI → http://127.0.0.1:5000 python web_ui/app.py # Run tests pytest pytest tests/test_critical_security.py -v
