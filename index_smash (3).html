01 / Overview

What is Amaterasu?

🧠
Autonomous Reasoning
Amaterasu is a self-directed AI agent that breaks down complex goals into actionable plans using Hierarchical Task Network (HTN) planning. It thinks, acts, observes and reflects — without constant human input.
🔄
Self-Improvement Engine
The agent can analyse its own code and generate patches in a secure staging environment. With 12 safety guarantees including rollback, kill-switch and rate-limiting, it improves itself responsibly.
⚡
Multi-Backend Routing
Intelligently routes tasks to the best LLM backend — Claude for deep reasoning, Groq for speed, Gemini for research, Ollama for local privacy. Semantic routing with sentence-transformers.
🛡️
Layered Security
8 security modules form a fortress: capability-based access control, circuit breakers, kill-switch, integrity checks, trust zones, outbound whitelisting, and a two-stage safety gate for all actions.
🕸️
Web UI & MCP
A Flask + Socket.IO web interface with 14 route modules provides real-time agent monitoring. The built-in MCP server makes Amaterasu available as a tool inside Claude Desktop and Cursor.
🌙
Dream Mode
When idle for more than 15 seconds, Amaterasu enters Dream Mode — analysing past episodes, recognising patterns, generating new task ideas, and optimising strategies automatically.
02 / Architecture

How It Works

The ReAct Loop

Amaterasu implements the ReAct pattern — Reason, Act, Observe — as its core execution loop. Every step is documented before execution, creating a transparent audit trail.

GOAL
GoalClarifier → ComplexityEstimator
Ambiguous goals are clarified via follow-up questions. Complexity is estimated to allocate the right reasoning budget.
PLAN
AdaptivePlanner → CheckpointPlanner
The HTN planner decomposes the goal into sub-tasks with dependencies. Checkpoints enable crash recovery mid-execution.
THINK
Fast Mind / Slow Mind (CognitionEngine)
Simple tasks: fast heuristic path. Complex tasks trigger DeepThinker + Tree-of-Thought + InnerDialogue for rigorous multi-step analysis.
SAFE
SafetyGate → Simulator (if risk ≥ 7)
Two-stage risk scoring: regex fast-pass → LLM deep-check. High-risk actions are simulated before execution.
ACT
ToolExecutor → EpisodicMemory
Tool is executed with adaptive timeout. Result is written to episodic memory for future learning.
REFLECT
ReflectionEngine → SkillEvolution
Outcome evaluated: continue / retry / replan / finish / escalate. Skills and strategies are updated based on performance.

Dual-Mind Cognition

Inspired by Kahneman's System 1/System 2 model, Amaterasu has two cognitive pathways:

⚡ Fast Mind
Pattern recognition and heuristics. Activates for simple, familiar tasks. Low latency, low cost. Complexity threshold: < 20 words, uncertainty < 40%.
🌀 Slow Mind
Activates when uncertainty > 40% or word count > 20. Engages DeepThinker, Tree-of-Thought (parallel strategy branches), InnerDialogue (pro/contra weighing), and MetaReasoner (quality check).

Reasoning Budget System

Simple (2 steps)Tier 1
Medium (6 steps)Tier 2
Complex (12 steps)Tier 3
Extreme (25 steps)Tier 4
Pipeline: Think → Simulate → Evaluate → Execute → Reflect → Learn

04 / Memory

Memory Architecture

Amaterasu uses a cascaded three-layer memory system inspired by human cognition. The MemoryOrchestrator queries all layers in priority order and fuses results by weighted relevance score.

⚡ Working Memory
Current task + last 50 actions in RAM. Fastest retrieval. Error entries get extra boost (×1.1) to prevent repeating mistakes.
Boost ×1.0
📖 Episodic Memory
All past actions persisted in SQLite with FTS5 full-text index. 10–100× faster similarity search than linear scan.
Boost ×0.8
🌐 Semantic Memory
Abstract consolidated knowledge: facts, concepts, learned relationships. FAISS vector store for semantic similarity. NumPy fallback available.
Boost ×0.6
🗄️ Knowledge Base
Structured long-term knowledge with category index. RAG integration feeds relevant knowledge into every LLM prompt.
Long-term

Memory Recall API

# Cascaded recall across all layers
from memory.orchestrator import MemoryOrchestrator

orchestrator = MemoryOrchestrator(
    working=loop.working_memory,
    episodic=loop.episodic_memory,
    semantic=loop.semantic_memory,
)
results = orchestrator.recall(
    "Python HTTP requests", top_k=5
)
context = orchestrator.build_context(
    results, max_chars=2000
)

Consolidation Pipeline

1
Episode Batch
Episodic memory accumulates experiences during operation.
2
Digest
Compact summaries are generated from episode batches in the background.
3
Consolidator
At shutdown, episodic → semantic memory consolidation. Important lessons survive; noise is filtered.
4
Deduper
Health report identifies duplicates and stale entries. GC removes them to keep memory efficient.
05 / Security

Security Architecture

8 security modules form a layered defence. Startup order is deterministic — each layer depends on the one below it being healthy before activating.

Capabilities
Cap enum controls access: SELF_IMPROVE, EXECUTE_CODE, NETWORK etc. require_capability(Cap.X) blocks unauthorised operations instantly.
Always On
Secrets Proxy
API keys never accessed directly in code. get_secrets_proxy().request_api_key("anthropic") — the proxy mediates all secret access.
Always On
Integrity Checker
SHA-256 baseline of core files established at startup. IntegrityChecker.verify() detects tampering with any monitored file.
Always On
Circuit Breaker
After N failures: OPEN state — all calls immediately blocked until cooldown expires. Separate breakers for Sandbox, Self-Improvement, and Plugin-Loader.
Always On
Trust Zones
CORE → PLUGINS → UNTRUSTED. assert_zone_flow(from, to) prevents unauthorised privilege escalation between zones.
Always On
Kill Switch
KillSwitch.activate(SafeMode.EMERGENCY) immediately blocks all LLM calls and self-improvement operations globally.
Always On
Outbound Whitelist
Monkey-patches requests.Session + urllib. Only LLM API domains are allowed. Everything else raises OutboundBlockedError.
Always On
Self-Rewriting
Disabled by default. Requires explicit self_rewriting: true in config. 12-step safe update pipeline with full rollback capability.

07 / Usage

Installation & Usage

1
Prerequisites
Python 3.11 or higher is required. Ollama is recommended for local LLM usage (no API key needed).
# Install Ollama from https://ollama.com, then:
ollama pull llama3
2
Install the Package
Clone the repo and install dependencies. The [all] extra includes all optional backends and features.
git clone https://github.com/yourname/amaterasu
cd amaterasu_v1

# Minimal installation
pip install -e .

# Full installation (recommended)
pip install -e ".[all]"
3
Configure API Keys
Copy .env.example to .env and fill in any API keys you want to use. Never commit keys to git.
# .env (never commit this file)
ANTHROPIC_API_KEY=sk-ant-...
GROQ_API_KEY=gsk_...
AMATERASU_API_TOKEN=your-web-ui-token
4
Launch Amaterasu
Multiple launch modes are available depending on your use case.
# Interactive CLI mode
python amaterasu.py --interactive

# Execute a single goal
python amaterasu.py --goal "Research GPU prices 2025"

# Autonomous mode
python amaterasu.py --autonomous --goal "Analyse my Python project"

# Web UI → http://127.0.0.1:5000
python web_ui/app.py

# MCP server (for Claude Desktop / Cursor)
python -m mcp.server
5
Run Tests
37 test files covering security, end-to-end flows, memory, contracts, backends, and more.
pytest
pytest tests/test_critical_security.py -v
pytest tests/test_e2e.py -v
pytest --cov=. --cov-report=term-missing
